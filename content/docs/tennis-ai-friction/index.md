Documentation can behave like a product interface when it helps the reader understand state, intent, limits, and next action.

This page establishes the documentation route, layout, and Content collection shape for the Tennis AI Friction project. The goal is not to hide unfinished architecture, but to make its edges readable.

## Goals

- Explain the project without pretending the backend exists yet.
- Keep technical context close to the editorial frame.
- Prepare a route tree that can grow page by page as the system becomes real.
